## NAPI-RS Binding Patterns

### Class wrapping
Rust structs use `#[napi(js_name = "SzEngine")]` to set the JS class name. The inner field holds `Box<dyn Trait>` or `Arc<SzEnvironmentCore>`. Non-constructor factory methods use `pub(crate) fn from_inner(inner: T) -> Self`.

### BigInt flag handling
All flags are `bigint` in JS. Rust methods accept `Option<BigInt>` and convert via:
```rust
let sz_flags = bigint_to_sz_flags(flags); // in flags.rs
// internally: flags.map(|f| SzFlags::from_bits_truncate(f.get_u64().1))
```

### Error mapping
Rust throws `napi::Error` with a `[CODE]` prefix in the message (e.g., `[SZ_NOT_FOUND] Record does not exist`). The JS wrapper parses this prefix and re-throws as the correct SzError subclass for `instanceof` support.

### Entry point structure
- `index.js` — auto-generated by napi-rs, handles cross-platform .node loading. DO NOT EDIT (napi build overwrites it).
- `sdk.js` / `configtool.js` — hand-written wrapper that loads `index.js`, wraps methods with error mapping, builds SzFlags, exports everything.
- `package.json` `"main"` points to `sdk.js` or `configtool.js`, NOT `index.js`.

### SzExportIterator
Implemented as a pure JS class in `sdk.js` (not in Rust). Wraps the native export handle with `Symbol.iterator` protocol. The `exportJsonEntityReport` and `exportCsvEntityReport` methods are overridden in JS to return `SzExportIterator` instead of raw handles.
