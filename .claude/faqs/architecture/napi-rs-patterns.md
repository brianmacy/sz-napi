## NAPI-RS Binding Patterns

### Class wrapping

Rust structs use `#[napi(js_name = "SzEngine")]` to set the JS class name. The inner field holds `Box<dyn Trait>` or `Arc<SzEnvironmentCore>`. Non-constructor factory methods use `pub(crate) fn from_inner(inner: T) -> Self`.

### BigInt flag handling

All flags are `bigint` in JS. Rust methods accept `Option<BigInt>` and convert via:

```rust
let sz_flags = bigint_to_sz_flags(flags); // in flags.rs
// internally: flags.map(|f| SzFlags::from_bits_truncate(f.get_u64().1))
```

The `get_all_flags()` function uses `Vec::with_capacity(80)` for pre-allocation.

### Error mapping

Rust throws `napi::Error` with a `[CODE]` prefix in the message (e.g., `[SZ_NOT_FOUND] Record does not exist`). The JS wrapper parses this prefix and re-throws as the correct SzError subclass for `instanceof` support.

The configtool crate uses a shared `json_serialize_error` helper in `error.rs` for JSON serialization errors across all modules.

The SDK error match uses a `_ => "SZ_UNHANDLED"` catch-all because `SzError` is `#[non_exhaustive]`.

### Entry point structure

- `index.js` — auto-generated by napi-rs, handles cross-platform .node loading. Committed to repo (required at runtime). Regenerated by `napi build`.
- `index.d.ts` — auto-generated TypeScript declarations from Rust source. Committed to repo.
- `sdk.js` / `configtool.js` — hand-written wrapper that loads `index.js`, wraps methods with error mapping, builds SzFlags, exports everything.
- `sdk.d.ts` — hand-written TypeScript declarations that re-export from `index.d.ts` plus error types, SzFlags, and SzExportIterator.
- `package.json` `"main"` points to `sdk.js` or `configtool.js`, NOT `index.js`.

### SzExportIterator

Implemented as a pure JS class in `sdk.js` (not in Rust). Wraps the native export handle with `Symbol.iterator` protocol. The `exportJsonEntityReport` and `exportCsvEntityReport` methods are overridden in JS to return `SzExportIterator` instead of raw handles. TypeScript types are in `sdk.d.ts` via module augmentation.

### Rpath embedding

The SDK `build.rs` detects the Senzing lib path (Homebrew on macOS, `/opt/senzing/er/lib` on Linux) and embeds it as an rpath via `cargo:rustc-cdylib-link-arg=-Wl,-rpath,<path>`. This eliminates the need for `DYLD_LIBRARY_PATH` or `LD_LIBRARY_PATH` at runtime.

### Dead code suppression

The configtool crate uses `#![allow(dead_code)]` because all functions are FFI exports invisible to Rust's dead code analysis. The SDK crate uses targeted `#[allow(dead_code)]` on specific FFI items only.
